Manifesting to be a SOC Analyst – Learning Phase 2

🛠️ Building Your First Home Lab

After understanding the fundamentals of what a SOC Analyst does in Phase 1, it's time to move from theory to practice. In this phase, we'll focus on creating a virtual home lab where you can safely simulate, analyze, and respond to cybersecurity incidents.

Why build a home lab? Because it gives you hands-on experience with tools like Splunk, Wireshark, and Sysmon without risking your main system. It's also the best way to practice incident triage, threat detection, and log analysis—skills every SOC Analyst needs.

🔧 What You’ll Need:

- A computer with at least 8GB RAM (16GB recommended)

- Virtualization software: VirtualBox (free)

- OS ISOs: Windows 10 and Kali Linux

🖥️ Setting Up Virtual Machines:

Install VirtualBox and create two VMs: one for Windows (victim) and one for Kali Linux (attacker). Ensure you set them up in an internal network mode so they can communicate with each other.

🔍 Tools to Install in Windows VM:

- Sysmon (system monitoring)

- Splunk (free version for log analysis)

- Microsoft Defender or any AV

💻 Tools in Kali Linux:

- Nmap, Wireshark, Metasploit, etc., for simulating attacks

🧪 Sample Activities:

- Generate failed login attempts and observe Sysmon logs

- Run a basic scan from Kali using Nmap and detect it in Splunk

- Simulate file changes or registry edits and see how they're logged

⚠️ Lab Safety Tips:

- Never connect your Kali VM to the internet unless needed and safe

- Use snapshots before testing malware or exploits

- Keep your testing network isolated

✅ Outcome of Phase 2:

By the end of this phase, you'll have a functioning home lab and practical experience analyzing security events. This is your mini SOC environment where you can practice what real analysts do!

Next in Phase 3, we’ll look into log analysis techniques and writing your first detection rules.
